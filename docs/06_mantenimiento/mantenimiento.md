# Mantenimiento del Sistema

## Objetivo

Establecer las acciones necesarias para mantener operativo, seguro y actualizado el Sistema Integral de Aula Virtual Docente.

## Tipos de mantenimiento

| Tipo | Descripción |
|---|---|
| Correctivo | Solución de errores detectados en módulos, formularios, APIs o base de datos. |
| Preventivo | Revisión periódica para evitar fallos futuros. |
| Adaptativo | Ajustes por cambios en requerimientos, currículo, roles o infraestructura. |
| Evolutivo | Incorporación de nuevas funcionalidades al sistema. |

## Actividades de mantenimiento

| Actividad | Frecuencia | Responsable |
|---|---|---|
| Revisión de errores del sistema | Semanal | Equipo de desarrollo |
| Verificación de conexión a base de datos | Semanal | Backend |
| Revisión de formularios | Mensual | Frontend |
| Actualización de dependencias | Mensual | Equipo técnico |
| Respaldo de base de datos | Semanal | Administrador |
| Revisión de pruebas funcionales | Después de cada cambio | Equipo de pruebas |

## Módulos sujetos a mantenimiento

- Login
- Dashboard
- Docentes
- Estudiantes
- Grados y secciones
- Asistencia
- Notas
- Sesiones
- IA Sesiones
- Reportes
- Configuración

## Conclusión

El mantenimiento del sistema permitirá garantizar la continuidad operativa, seguridad, estabilidad y mejora progresiva del software.
